# OKX 交易员名额监控器 (抢位置脚本)

## 为什么需要这个？
很多顶级的带单员（Trader）名额（500/1000人）常年是满的。
但是，**跟单用户是流动的**。每天都有人取消跟单、或者因为亏损离场。
这些空出来的名额往往在几分钟甚至几秒钟内被抢光。
手动刷新页面是不可能抢到的，你需要一个脚本 24 小时帮你盯着。

## 如何使用

### 1. 找到目标交易员 ID
在浏览器打开你想跟单的交易员主页，URL 里通常有一串字符，比如：
`https://www.okx.com/copy-trading/trader/ABC123456`
那个 `ABC123456` 就是 ID。

### 2. 配置脚本
打开 `monitor_slots.py`，修改以下几行：

```python
TRADER_ID = "把这里的 ID 换成真实的"
EMAIL_USER = "你的邮箱@gmail.com"
EMAIL_PASSWORD = "你的应用专用密码"
EMAIL_TO = "接收通知的邮箱"
```

### 3. 运行
在终端执行：
```bash
python monitor_slots.py
```

一旦发现空位，它会立刻给你发邮件。收到邮件后，**立刻**打开 App 手动跟单。

---

## 进阶玩法：全自动跟单 (需要编程基础)
如果你想真正做到“秒抢”，需要在发现空位时直接调用 OKX API 进行跟单。
但这涉及到你的账户 API Key 权限（交易权限），风险较大，建议先用邮件提醒。

## 关于“不跟单也能看持仓”
如果交易员设置了“隐藏持仓细节”，神仙也看不到。
如果他是公开的，可以使用 `monitor_positions_concept.py` 里的思路，用 Selenium 模拟浏览器去爬取。但要注意 OKX 的反爬虫很严，容易封 IP。
最稳妥的办法还是抢名额。
